
## `<input>` 태그 사용하기
-  입력 받을 정보에 따라 여러 가지 type 속성이 있습니다. 제가 소개하는 것 이외에도 엄청 많아서 추후 mdn 공식 문서를 확인해보시는 것을 추천드립니다.
### 태그 관리하기

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <form action="#">
    <label for="txtIP">text</label><br>
    <input 
      id = "txtIP"
      type="text"
      placeholder="5자 이하"
      maxlength="5">
      <br><br>
      <label for="pwId">pw</label><br>
      <input
        id="pwID" 
        type="text"
        placeholder="최소 4자"
        minlength="4"
        >
      <br><br>
      <label for="searchId">serach</label><br>
      <input
        id="searchId" 
        type="text"
        >
      <br><br>
      <label for="telId">tel</label><br>
      <input
        id="telId" 
        type="text"
        >
      <br><br>
      <button type="submit">제출</button>
  </form>
  
</body>
</html>

```

이상의 html에서 tyoe으로 1. text / 2. password / 3. search / 4. tel 을 사용했다.
input 태그의 default는 text로 생성된다.

1. text
  - maxlength / minlength를 확인할 수 있다. 이때 안내문을 출력해주는 것이 사용자 입장에서 좋을것이다.
  이때 사용할 수 있는 속성이 placeholder가 되겠다.
2. password
  - 우리가 무엇을 입력하는지 모르도록 글자가 점(별)형태로 가려진다.
  - button - type=submit이었을때 4자 미만으로 입력했을때 안내문 / 경고문이 출력되었습니다. 이는 브라우저마다 나타나는 방식이 다르기 때문에 js등을 이용하여 추가 경고 팝업을 띄우는 방식을 주로 사용한다.
3. search 
  - 다른 거 별로 차이 없어 보이는데 텍스트를 입력하면 입력 폼 오른쪽에 x표시가 되고, 이거 클릭하면 입력 한 텍스트를 초기화해 준다.
  - 근데 이것도 css로 처리하는 편
4. tel
  - 전화번호 입력 폼. 웹에서는 별 차이없고 모바일에선 번호판 키패드가 팝업된다.

### 숫자 관련 `<input>`
04_numberinputtag

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f"><!-- 수량조절할때 쓸 수 있는거, max를 남은 재고만큼만 가져오게 해야겠다. -->
  <h1>숫자 관련 인풋 태그</h1>
  <label for="numIP"></label><br>
  <input 
    id="numIp" 
    type="number" 
    min="0"
    max="10">
  <br><br>

  <label for="rangeIP"></label><br><!-- 볼륨조절할때 쓸 수 있는거 -->
  <input 
    id="rangeIP" 
    type="range" 
    min="0"
    max="100"
    step="20"><!-- 20씩 움직인다. 즉, 세세한 조절 불가 -->
  <br><br>

  <label for="dateIP"></label><br><!-- 볼륨조절할때 쓸 수 있는거 -->
  <input 
    id="date" 
    type="date" 
    min="2010-01-01"
    max="2030-12-31">
  <br><br>
</body>
</html>
```

1. number - 입력 폼에 숫자만 넣을 수 있음. 화살표 생김. 화살표 모양도 브라우저에 따라 다라므
2. range - 슬라이드 바를 왼쪽 오른쪽으로 조절하는 타입. step  속성에 숫자 단위를 입력하는데, js와 함께 사용하는 편
3.  date - 달력 모양이 나오는데 브라우저마다 다르게 나옴. react등에서 사용하는 편

4. disabled : 사용자가 값을 입력할 수도 없고 결과가 전송되지도 않음.
5. required : 반드시 결과값을 입력해야하는 것을 의미한다.

### `<textarea>`
- input 태그와 달리 긴 분량의 텍스트, 즉 텍스트 입력 영역이 페이지에서 여러줄 이상을 차지하도록 할 때 사용하는 태그
- cols 속성 (너비) / rows 속성 (줄 수)를 통해 텍스트 영역의 크기를 설정함
- cols="64" / rows="5"로 한 번 만들어봐라.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>
  <h1>textarea 태그</h1>
  <label for="message">메시지를 입력하세요</label><br>
  <textarea id="message" cols="64" rows="5">여기에 기본값을 입력할 수 있다.</textarea>
</body>
</html>
```

- 주의사항
  - textarea 태그의 경우 input 태그의 여러줄 버전이라고 생각하기 쉽고, 그게 사실이 맞다.
  하지만 value=""를 통해 default 값을 입력할 수 있는 input 태그와 달리 여는 태그/ 닫는 태그로 구성되어있기 때문에 내부 요소로 default 값을 입력할 수 있었다.

### `<option>`
09_optiontag.html
```html
<!DOCTYPE html>
<html lang="en">

  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

<body bgcolor="5f5f5f">
```

1. select + option 태그 : 사용자 눈에 보이는 결과와는 별개로 value 속성을 이용하여 전송될 결과값을 추가해줘야한다.
그래서 아무것도 선택하지 안흔 상태인 '--언어 선택--'에는 value=""으로 처리
  - default 값으로 무언가 하나를 보여주고 싶다면 option 태그내에 selected 속성을 지정해 줄 수 있다.
  - select 내에 multiple을 지정할 수도 있다.
  - shift 키 / ctrl 키를 통해 다중 선택이 가능하긴 하지만 별도의 설명이 없다면 사용자가 알아채기 힘들기 때문에 해당방식은 실무에서는 거의 사용되지 않고,
  차라리 그럴거면 input 태그의 checkbox 속성을 이용해서 그냥 여러개 선택하는게 낫다.
2. select + optgroup 태그 :
  - 옵션 안에 있는 내용들을 그룹화하는 기능, optgroup에 있는 내용은 선택이 불가능하다.
3. datalist 태그 :
  - label + input 태그로 동일하게 보이지만 그 밑에 datalist 태그가 있다.
  datalist 내의 option 태그들을 추천 답안을 미리 넣어둔 것이라고 할 수 있다. 이 때 중요한 점은 label의 for 속성과 input의 id 속성을 일치시켰고, input의 list 속성과 datalist의 id 속성을 일치 시켰다는 점이다.

### 정도를 표현하는 태그

1. progress 태그에서 시작 버튼을 클릭하면 뭐 진행되거나 하는데, js 와의 연동성을 통해서 작동한다.
2. 내부에 max 속성이 있는데, 기본값 =1/ 최대값 = 100에 해당한다. 여기서 value 속성과 합쳐서 0,50,70,100 등으로 바꾸게 되면 바의 진행 정도가 바뀌게 된다.

3. meter 태그의 경우 변화가 없는 고정된 진행 상태를 나타낼 때 사용한다.
  - progress와의 차이점은 min 속성이 존재한다는 점. progress는 어차피 최소값이 0이기 때문에 필요없는데 , 
  meter의 경우 음수값을 지정할 수도 있다. 그렇기 때문에 meter 사용시에는 min/max를 둘다 지정해주는것이 좋다.

  - low / high 속성은 상대적인 값
  - optimum 속성을 통해 이상치를 지정해 준다. optinum이 50일 경우
  value가 20이면 좋지 않은 수치, 50이면 이상적인 수치, 그리고 80이면 좋징 ㅏㄶ은 수치에 해당하기 때문에
  노란색 / 초록색 / 빨간색이 된다.

### 아무것도 하지 않는 `<div>` 태그 vs. `<span>` 태그
- 이전까지의 학습에서는 태그들이 각기 다른 역할이 존재했었지만, 이번에는 아무 역할도 하지 않는다면 태그에 대해 학습한다.
- 하지만 css 적용시에 있어서 매우 중요하기에 성격의 차이를 학습하겠다.

`<span>` 태그와 `<div>` 태그는 기능적으로는 아무것도 하지 않는다.
- 차이점
  1. `<span>` 태그는 텍스트 내용이 들어간 부분만큼만 영역을 차지 : 인라인
  2. `<div>` 태그는 텍스트가 포함된 브라우저 가로 전체 영역을 차지 : 블록

- 즉, 여태 배워온 모든 태그들은 기본적으로 인라인이거나 블록이거나 하는 성격을 지니고 있는 상태
- `<b>` 태그는 inline이다.
- `<p>` 태그는 block이다.

```html
안녕하세요 <b>홍길동</b> 입니다.
안녕하세요 <p>홍길동</p> 입니다.
```
