# 태그 속성 사용하기

## 태그의 속성(특성)
- html 태그의 속성을 영어 원문 상 python과 동일하게 attribute를 사용
그런데 mdn 문서에서 전에 말했던 것처럼 특성이라고 번역하는 중인데, 현재 국내의 대부분의 강의에서 속성이라고 표기하고 있고 현직에서도 그러는 중이라서 속성이라는 표현을 중심으로 강의한다.

- 예시
    1. `<html lang="ko">`
    2. `<img src="" alt="">`
이상의 예시에서 img 태그는 그림을 넣어주는 태그인데, 원본 파일명이 src속성에 들어가고, 만약 그림 파일이 없을 경우 대신 표시할 alt(alternative)속성이 존재한다.
근데 이 img 태그 내의 src, alt 속성은 p태그나 h1 태그에는 존재하지 않는다. 즉, _태그의 동작을 설정 및 제어하는것_ 이 **태그의 속성**이며, 각 태그마다 설정 가능한 속성들이 있다.

### 인용문 관련 태그
특정 텍스트를 외부에서 가져와 인용하거나 참조할 때 사용하는 태그
1. `<blockquote>`

```html
<div>
  <blockquote cite="https://www.huxley.net/bnw/four.html">
    <p>
      Words can be like X-rays, if you use them properly—they’ll go through
      anything. You read and you’re pierced.
    </p>
  </blockquote>

</div>
```

<>는 태그로 보일 수 있으니 &lt;와 &gt;를 대신 쓴다.
`&lt;blockquote&gt; = <blockquote>`

* 참조
- 어떤 태그를 작성했을 때, img 태그의 경우는 default로 src와 alt 속성이 자동 생성되었지만, 아닌 경우도 있다.
그런데 특정 속성이 태그 생성시에 꼭 포함되었으면 좋겠다면 사용할 수 있는 emmet이 존재 한다.
`blockquote[cite` 까지 입력하고 tab을 했을때 cite속성도 자동완성 된다.

그리고 less than/ greater than 부분을 설명하자면
`<>`를 입력하고자 할때, .html파일 내에서 작성하면 tag 이름으로 인식하기때문에 브라우저에는 출력이 안된다.
이때 부등호를 다른 대체 텍스트로 입력해주는 작업이 필요한데, 이를 엔티티 이름이라고 한다.
* 예시
    1. `<` : &lt - less than
    2. `>` : &gt - greater than
    3. ` ` : `&nbsp;` -none breaking space

- blockquote 최종 정리
    - 대표적인 특징으로 인용 내용을 들여쓰기로 표현해준다. 근데 두세줄 이상일 때 가치가 있다.
    - 이상을 이유로 짧은 텍스트의 경우 `<q>`를 사용할 수 있다. quote의 축약어
    인용이라 cite속성도 가지고 있다.

# 태그로 여로 요소 나열하기
- 어떤 정보를 나열할 때 목록으로 표기하면 시인성이 높아진다.
이런 목록을 표기할 때 순서가 없는 태그와 순서가 존재ㅔ하는 태그로 나뉜다.
## 목록을 표현하는 태그
- `<ul>` 태그는 unordered list의 축약어로 순서가 상관없는 리스트를 만들 때 사용한다.

```html
<ul>
    <li> 딸기 </li>
    <li> 사과 </li>
    <li> 포도 </li>
</ul>
```

- `<ol>` 태그 알아서 번호가 매겨진다.

```html
<ol>
    <li> 딸기 </li>
    <li> 사과 </li>
    <li> 포도 </li>
</ol>
```

이상의 두 경우에서 `<li>`태그는 list item의 축약어로 두 태그의 하위 항목을 표기할때 사용한다.

특히 `<ol>`의 속성중 type이라는 것이 있는데, 이를 이용해 리스트의 순서가 아라비아 숫자로 쓰일지, 다른것으로 쓰일지 지정이 가능하다.
    - type = "A"로 지정하면 ABCD
    - 디폴트는 아라비아 숫자
    - start로 시작하는 문자나 수를 지정할 수 있으며, 0이 아닌 1부터 시작한다.

## 용어와 정의를 나열하는 태그
어떤 리스트를 1:1로 매칭하는 경우가 있다고 가정하겠다.
용어:정의가 1:1 대응도 가능하고, n:n도 가능할것이다.
즉, 여로 용어에 하나의 정의를 붙어거나 하나의 단어에 여러 정의가 있을 수 있다.

- `<dl>` : 정의 목록
- `<dt>` : 정의할 용어
- `<dd>` : 축약어

## `<form>` 태그 사용하기
- 사용자로부터 입력을 받기 위한 양식을 작성하는 태그를 통츨어서 지칭

### `<form>` 태그의 속성

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>01_formtag.html</h1>
  <form action="./01_formtag.html" method="get">
    <label for="name">이름</label>
    <input id="name" type="text">
    <br><br>
    <label for="age"></label>
    <input id="age" name="my-age" type="number">
    <br><br>
    <button type="submit">제출</button>
    <button type="reset">초기화</button>
  </form>
</body>
</html>
```

- 이상의 예시 코드에서 사용되는 속성들
    1. `<form>` 태그
        - action
        - method
            - 현재 수준에서 경험해보지는 못했지만 우리가 이름과 나이를 입력했을 때
                그 정보를 가져가서 보여주는 결과 페이지에 해당한다. 현재 수준에는 form 태그 내에
                action / method 속성이 있다는것만 알아도 된다.

    2. `<input>` 태그
        - 이건 다른 챕터에서 설명예정
        - name : HTML에서 특별한 의미를 갖는게 아니라
        서버에서 각 항복을 받을 때 무엇에 대한 값인지 구분할 수 있도록 하는 용도에 해당
        DB와의 연결성과 관련

    3. `<label>` 태그 : input 태그와 유기적으로 연결됨. input 태그에 부여한 id 속성값을 label 태그에서의
    for 속성에서 불러오기 때문이다.

    4. `<button>` 태그 : 
        - type - submit / reset으로 사전에 정의되어 있었다.
        submit의 경우 입력한 결과값을 최종 서버나 특정 페이지 주소로 보내는, 즉 제출하는 용도<br>
        reset은 초기화
