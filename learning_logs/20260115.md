## 웹 디자이너 vs. 웹 퍼블리셔
웹 디자이너가 작업한 결과물을 HTML + CSS 문서로 작성하는 사람을 우리나라에서는 웹 퍼블리셔라는 표현을 쓴다.

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>네</title>
</head>
<body>
    <h1>Hello html</h1>
    <p>
        Gazuaaaa<br>
        ㅇㅇㅇㅇㅇ<br>
        <h2></h2>
    </p>
</body>
</html>
```

!+tab으로 자동 생성되는 것은 이상과 같다.
이는 웹 개발 시에 기꼐 또는 시스템에 전달하는 각종 정보로, HTML 문서 작성시에 꼭
필요한 내용들을 담고 있다. 문서 제목, 작성자, 사용언어 및 각종 메타 정보등으로 구성되어 있는데,
어차피 형식자체는 동일하기 때문에 단축키를 지원한다.
이를 Emmet이라고 한다.

- <html lang="en">의 의미
lang 속성은 HTML 문서의 웹사이트가 어떤 언어로 구성되어있는 컴퓨터에게 알려주는 역할을 한다.
이것을 지정해줬을때 구글 / 네이버 등에서 검색할때 한글로 된 웹사이트를 찾는 것이 가능하다.

- SEO : Search Engine Optimization

## 텍스트 관련 태그
웹 페이지에서 텍스트를 지정할 때는 내용에 따라 글자를 강조하거나 증요하다고 표시하게 됩니다.
나무위키 같은데 보면 뻘글 있는쪽에 취소선 있고 그런데 그렇게 텍스트의 스타일을 다양하게 조절하는 태그들도 존재한다.

### 스타일 강조 태그
제목이 아니라 본문 내에서의 세부 스타이 ㄹ관련
1. `<b>`  볼드체
2. `<strong>` 굵게
3. `<i>` 이탤릭체
4. `<em>` 기울임체

- 옛날에는 HTML과 cSS가 분리되지 않았기 때문에 웹 페이지 상에서의 서체를 굵게 지정하거나 기울이는 것을 전부 다 태그로 처리 했었다.

- 5 버전 이후부터는 태그는 코드적으로 어떤 위상에 속하는지만 표시하고 데코는 CSS로 넘어간다.

- 즉, b태그와 i태그는 시각적으로 강조하는 효과, strong과 em은 text들의 중요 토픽에 해당하는 부분이란것을 컴퓨터에게 알려주는 역할을 하낟.

- 그러면 검색 엔진에서 특정 검색어를 지정할때 해당 부분에서 동일한 부분이 있는지를 체크하게 된다.

### 첨자 태그
위 첨자 및 아래 첨자라는 개념을 학습한 적이 있다.
보통은 영어시간이나 수학시간에 배운다.
- 위 첨자 예시
`<sup>` 1<sup>st</sup>  
- 아래 첨자 예시
`<sub>` 1<sub>st</sub>

### 밑줄 태그와 취소선 태그
_이탤릭체_ 쓰거나 **이렇게** 강조하는 방식이 markdown에 쓰인다(notion에서도) 

밑줄 예시
<u>`<u></u>`</u>

취소선 예시
<s>`<s></s>`</s>
